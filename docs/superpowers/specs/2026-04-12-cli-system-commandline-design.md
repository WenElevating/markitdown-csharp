# CLI Enhancement: System.CommandLine Integration

## Goal

Replace the hand-rolled argument parser with `System.CommandLine` to provide proper `--help`, `--version`, multi-file support, LLM configuration, and format listing.

## Current State

- `CliRunner.cs` has a manual parser that iterates args with a for-loop
- `--help` throws an `ArgumentException` caught as an error (exit 1)
- No version info, no format listing, no LLM config, single-file only

## Design

### Command Structure

```
markitdown <path> [<path>...] [options]
```

One or more input paths (files or URLs). When multiple paths are given, `-o` must point to a directory.

### Options

| Option | Short | Description |
|--------|-------|-------------|
| `--output` | `-o` | Output file or directory (default: stdout) |
| `--version` | `-V` | Show version number |
| `--list-formats` | | List all supported formats |
| `--llm-key` | | OpenAI API key (enables LLM captioning) |
| `--llm-model` | | LLM model name (default: `gpt-4o`) |
| `--llm-endpoint` | | Custom API endpoint (Azure OpenAI) |
| `--help` | `-h` | Show help (auto-generated by System.CommandLine) |

### Behavior

**Single file + no `-o`:** Write Markdown to stdout.
**Single file + `-o file.md`:** Write to file.
**Multiple files + `-o dir/`:** Each file outputs as `dir/{filename}.md`.
**Multiple files + no `-o`:** Error — `-o` is required for multi-file mode.
**No arguments:** Show help (exit 0).
**`--version`:** Print version from assembly, exit 0.
**`--list-formats`:** Enumerate all registered converters and their extensions, exit 0.
**`--llm-key` provided:** Create `OpenAILlmClient` and inject into `DocumentConversionRequest.LlmClient` for each conversion.

### Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Input error (file not found, unsupported format) |
| 2 | Conversion error |
